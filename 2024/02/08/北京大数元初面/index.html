1、final/finally/finalize的区别

final 关键字：

final 是一个关键字，用于声明类、方法或变量。其含义取决于其应用的上下文：

  • 当用于类时：final class ClassName，表示该类是不可被继承的，即它是一个最终类，不能有子类继承它。
  • 当用于方法时：final returnType methodName()，表示该方法是不可被子类重写的，即它是一个最终方法，子类不能覆盖该方法的实现。
  • 当用于变量时：final dataType variableName，表示该变量是一个常量，一旦赋值后就不能再改变其值。
finalize() 方法：

finalize() 是 java.lang.Object 类中的一个方法，它用于在对象被垃圾回收之前执行清理操作。当一个对象变得不可达时，垃圾回收器会调用对象的 finalize() 方法来释放资源或执行其他清理任务。然而，自从 Java 9 开始，finalize() 方法已经被标记为废弃（Deprecated），不再推荐使用，而是推荐使用其他显式资源管理的方式，如 try-with-resources 或 java.lang.ref.Cleaner。

finally：

finally作为异常处理的一部分，它只能用在try/catch语句中，并且附带一个语句块，表示这段语句最终一定会被执行（不管有没有抛出异常），经常被用在需要释放资源的情况下。

2、having和where的区别

having是在分组后（group by）对数据进行过滤

where是在分组前（group by）对数据进行过滤

having后面可以使用聚合函数

where后面不可以使用聚合

在查询过程中执行顺序：from>where>group（含聚合）>having>order>select。

5、threadlocal的原理,使用threadlocal需要注意的点

  • ThreadLocal本质来说就是一个线程内部存储类，从而让多个线程只操作自己内部的值，从而实现线程数据隔离，在ThreadLocal中有一个内部类叫做ThreadLocalMap一个属性table存储数据。

  • 每一个Thread维护一个ThreadLocalMap，在ThreadLocalMap中的Entry对象继承了WeakReference。其中key为使用弱引用的ThreadLocal实例，value为线程变量的副本，强引用，并不会被回收，强烈建议：务必手动remove

12、NGINX反向代理实现过程？nigix出错时候的log两个，区别，如何进行排查？反向代理的配置详细说一下。

1）（苍穹day01）代理服务器来接收网络上请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给网络上连接的客户端。

优点：提高访问速度，进行负载均衡，保证后端服务安全

2）日志区别

①访问日志（Access.log）

记录每一次HTTP请求的访问状态。时间，请求方式，状态码

②错误日志（Error.log）

记录Nginx处理HTTP请求的错误状态以及Nginx本身服务的运行错误状态。时间，请求路径，报错原因，中文路径错误，应该安装在英文路径下。
